# Decisions

这里维护仓库级决策与文档治理规则。标准是“这条信息是否会影响后续 agent 的判断或实现路径”；如果不会，就不要放在这里。

## 当前有效规则

- `2026-03-08`：`docs/` 改为 agent-first 四层结构：`context/`、`runbooks/`、`decisions/`、`execplans/`。原因是原目录把稳定背景、排障手册、审计归档和活计划混在一起，增加 agent 检索成本。
- `2026-03-08`：`docs/` 下文件名统一改为 ASCII `kebab-case`。原因是便于跨平台处理、批量检索和脚本消费；中文保留在标题与正文。
- `2026-03-08`：`docs/README.md` 成为 `docs/` 唯一总入口。原因是 agent 不应通过猜测或历史习惯决定从哪个文档进入。
- `2026-03-08`：只保留最小活文档集合：`docs/execplans/*.md`、本文件、`docs/CHANGELOG.md`。原因是仓库主力协作者是强 agent，文档体系应优先降低判断成本，而不是增加维护负担。

## 记录规则

- 只记录仓库级决策，不记录单任务实现经过。
- 每条记录至少写明日期、决策内容和原因。
- 如果某项决策失效，直接在这里追加废止或替代说明，不删除历史。
