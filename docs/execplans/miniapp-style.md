# 小程序前端美术风格定位说明

根据我们目前迭代的设计稿，本项目的小程序美术风格定位为“专业高效、低干扰、B2B 工具化”。它摒弃了传统 C 端电商（如淘宝、京东）那种高饱和度、高促销感的设计，转而追求一种类似于 SaaS 或企业管理后台的沉浸式办公体验。

以下是详细的美术风格总结：

## 1. 核心视觉语言 (Core Visual Language)

- 设计原则：“信息密度优先，装饰性减法”。B2B 用户更在乎规格、阶梯价、货期和订单状态，因此 UI 布局极为克制，确保用户在每一屏都能快速捕捉到核心数据。
- 风格流派：属于 “Modern Enterprise UI (现代企业级 UI)”，融合了简约主义与结构主义，强调功能逻辑。

## 2. 色彩系统 (Color Palette)

- 主色 (Primary)：采用商务深蓝（#1677FF 或类似深蓝色）。蓝色代表信任、稳定与专业，常用于关键按钮（如“提交意向单”）、链接以及选中状态的图标。
- 辅助色 (Secondary)：
  - 警示类：使用克制的橙色或红色，仅用于“待处理订单”或“议价入口”，作为视觉引导，而非大面积装饰。
  - 阶梯价强调：使用极浅的灰蓝色背景块来包裹价格表格，增加阅读舒适度。
- 背景与中性色：大面积使用纯白（#FFFFFF）或超浅灰（#F5F7FA）。文字采用深灰色（#333333）和中灰色（#666666），通过字阶大小而非颜色来区分信息层级。

## 3. 布局与组件 (Layout & Components)

- 组件库基因：视觉表现高度兼容 Taroify（Vant 的 Taro 版本）。
- 单元格 (Cell)：采用无边框或极细分割线设计，增加呼吸感。
- 标签 (Tags/Chips)：规格选择采用圆角矩形的小标签，替代复杂的选择框，提高 SKU 选择效率。
- 卡片 (Cards)：去掉复杂的投影（Shadow），改为使用浅灰色细边框，减少视觉负担。
- 留白 (Whitespace)：在“我的”页面和“详情”页面中，增加较大的垂直间距，防止多规格、多价格信息堆砌带来的压抑感。

## 4. 图标与字体 (Icons & Typography)

- 图标风格：采用线性图标 (Line Icons)。线条粗细一致（通常为 1.5px 或 2px），给人以精准、标准化的工程感。
- 字体规范：优先使用系统默认字体（PingFang SC/Microsoft YaHei）。标题加粗以区分区块，正文保持常规粗细，核心数字（如价格、订单金额）通过半等宽字体展示，确保排列整齐。

## 5. 业务场景的特殊处理

- 意向驱动：“加入购物车”和“议价”按钮通常并排平分底部空间，或通过颜色权重（一深一浅）引导用户，体现了非纯电商的业务独特性。
- Excel 逻辑：导入页面的设计更像“数据看板”，使用进度条和成功/失败状态图标，强调任务的完成感而非购物的快感。

## 总结

这套风格可以概括为：“冷静、高效、克制”。它旨在让采购员在处理繁杂物料单时，感觉到这是一个可靠的生产力工具，而不是一个诱导消费的娱乐平台。
